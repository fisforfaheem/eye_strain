# EyeGuard

EyeGuard is a Flutter app that lets a user sign in, take a selfie, run a basic eye-strain check with Google ML Kit, and save the result to history. It uses Firebase Auth for login, Firestore for per-user records, and local storage for the captured image/history files kept on the device.

## Why I Built This

I wanted a small app that could test the idea of checking for eye strain from a quick front-camera photo instead of just using a fixed timer. The main goal here was getting the full flow working end to end: auth, camera capture, analysis, result screen, and saved history.

## Features

- Email/password sign up and sign in with Firebase Auth
- Front camera capture from the home screen
- Eye analysis using Google ML Kit face detection
- Break / no-break result based on eye openness and a few simple checks
- Saved history per user
- Local image/history persistence on device
- Basic profile screen with sign out and local history clearing

## Getting Started

This repo is currently set up around the Android flow. Firebase is configured for Android and web in the repo, but the app code itself uses mobile file/camera APIs, so Android is the main target right now.

### Prerequisites

- Flutter SDK
- Dart SDK
- Android Studio or a working Android device/emulator
- A Firebase project with Auth and Firestore enabled

### Run locally

```bash
git clone <repo>
cd eye_strain
flutter pub get
flutter run
```

### Firebase setup

The project already includes Firebase client config files, but if you want to run it against your own Firebase project, update these:

- `lib/firebase_options.dart`
- `android/app/google-services.json`
- `firestore.rules`

You’ll also want:

- Email/password auth enabled in Firebase Auth
- Firestore created in production mode
- The Firestore rules in this repo deployed

## Project Structure

- `lib/main.dart` bootstraps Firebase and the app
- `lib/screens/auth/` has the login/register screen
- `lib/screens/home/` has the camera flow and analysis UI
- `lib/screens/history/` shows saved checks
- `lib/screens/profile/` has account-related actions
- `lib/services/` contains auth, face detection, and eye-strain logic
- `lib/models/` contains the `EyeCheck` model
- `android/`, `ios/`, `web/`, `macos/`, `windows/`, `linux/` are the platform folders generated by Flutter

## Usage

1. Create an account or sign in.
2. Open the home screen.
3. Take a photo with the front camera.
4. Wait for the analysis result.
5. Check the saved history if you want to review past scans.

## Notes

- The detection logic is intentionally simple right now. It is more of a working prototype than a medical tool.
- Captured images are copied into local app storage and the result metadata is also saved to Firestore.
- Firestore access is locked down per user in the rules file, so users can only access their own records.
- Android is the only platform that is fully wired up in the current repo. Other platform folders exist, but Firebase options are not finished for all of them.
